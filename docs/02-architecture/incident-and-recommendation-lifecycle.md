# Incident and Recommendation Lifecycle

## Incident states

Detected -> Triaged -> Evidence Collecting -> Investigating -> Recommendation Ready -> Awaiting Approval -> Executing -> Verifying -> Resolved -> Learned.

## Recommendation states

Draft -> Challenged -> Policy Checked -> Ready for Review -> Approved | Rejected | Needs Changes -> Scheduled -> Executing -> Succeeded | Failed | Rolled Back -> Verified -> Scored.

## Mandatory recommendation content

- Problem statement.
- Scope and affected assets.
- Evidence.
- Confidence and uncertainty.
- Alternatives.
- Risk and blast radius.
- Preconditions.
- Backup or snapshot guidance.
- Execution plan.
- Rollback plan.
- Downtime expectation.
- Approval requirements.
- Verification checks.
- Expiration time for stale recommendations.
