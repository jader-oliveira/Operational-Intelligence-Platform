# Pilot Plan

## Recommended pilot

Select one non-production or lower-risk VMware cluster supporting a clearly owned business service.

## Entry criteria

- Read-only API credentials.
- Metrics and logs available.
- Change and incident history available.
- Named subject-matter experts.
- Success baseline captured.
- Data classification approved.

## Pilot stages

1. Inventory and coverage.
2. Dependency validation.
3. Historical incident replay.
4. Shadow incident investigation.
5. Recommendation review without execution.
6. Controlled approved remediation in test.
7. Outcome and trust assessment.

## Exit criteria

Coverage above 95% for selected assets, evidence-backed investigation on agreed incident classes, no critical permission bypass, measurable accuracy above baseline, and engineer acceptance of workflow.
